Solana基础-Token解析&创建

Solana上的代币

  • 代币是代表对各种资产所有权的数字资产。 代币化使得财产权的数字化成为可能，是管理 可替代和不可替代资产的基本组成部分。
    • 可替代代币代表同类型和同价值的可互换和可分割资产(例如 USDC)。
    • 不可替代代币(Non-Fungible Token，NFT)代表不可分割资产的所有权(例如艺术品、音乐、视频等)。

SPL（Solana Token Library）

  • Token Program（代币程序）是Solana上用于管理代币的智能合约。
  • Mint Account（铸币账户）是用于存储代币信息的账户。
  • Token Account（代币账户）是用于存储代币余额的账户。
  • Associated Token Account（关联代币账户）

Mint Account （铸币账户）

Mint Account 是每个SPL Token的核心帐户，负责代币的定义和管理。
它包含了代币的全局属性，比如总供应量和小数位数。
每个代币都由一个独立的Mint Account来管理。

  • 属性

    • Decimals（小数位数）：定义代币的最小单位，通常是0到9之间的整数。
    • Supply（供应量）：代币当前的总供应量
    • Mint Authority（铸币权限）：可以铸造新的代币的帐户
    • Freeze Authority（冻结权限）：可以冻结或解冻代币帐户的权限（可选）

  • 功能

    • 铸造代币： 当Mint Authority执行铸币操作时，新的代币增加到总供应量中，并分配给指定的Token Account。
    • 销毁代币：减少总供应量

Token Account（代币账户）

Token Account 时用于持有特定SPL Token的帐户。
每个用户或者合约如果想要持有某种SPL Token，必须有一个与该Token对应的Token Account。
不同的代币有不同的Token Account，帐户之间无法混用。

  • 属性

    • Amount（余额）：帐户中持有的代币数量
    • Owner（帐户拥有者）：控制该帐户的用户或合约地址。
    • Mint（铸币帐户关联）：该帐户与哪个Mint Account关联。
    • Delegate（代理帐户）：可以被授权管理该帐户的其他帐户（可选）
    • State（状态）：帐户是否出于冻结状态

  • 功能

    • 接受和发送代币：Token Account 可以接受其他帐户的代币，并通过转账指令将其发送给其他帐户。
    • 代理权限管理：可以设置Delegate权限，允许其他帐户管理该Token Account。

Associated Token Account（关联代币账户）

ATA（Associated Token Account）是一个特殊的Token Account，它简化了SPL Token的帐户管理。
ATA是自动与钱包地址绑定的帐户，每个钱包地址和每个代币地址的Mint对应一个唯一的Token Account。因此该类帐户类型极大简化了代币的管理。

  • 特点:
    • 唯一性:每个钱包地址只能有-个和某个 Mint 关联的 ATA。
    • 自动生成 ATA，开发者不需要手动管理自动生成:Solana提供了工具账户。
  • 功能:
    • 自动处理钱包地址与代币账户之间的关联，方便用户管理不同的 SPLToken.
