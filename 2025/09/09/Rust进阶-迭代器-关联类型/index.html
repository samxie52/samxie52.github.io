Rust进阶-迭代器&关联类型

发表于 2025-09-09 阅读次数：
Rust进阶-迭代器&关联类型（迭代器的定义、迭代器的使用、关联类型）

什么是迭代器

  • 迭代器模式：对一系列想执行某些任务
    -迭代器负责：

    • 遍历每个项
    • 确定序列（遍历）何时完成

  • Rust中的迭代器：

    • 懒惰的：除非调用消费迭代器的方法，否则迭代器本身没有任何效果。

迭代器的实现

  • 迭代器是一个能够逐一生成元素的对象。它提供了一个统一的接口。用于遍历容器中的元素，同时保证了类型安全和内存安全。

  • 在Rust中，迭代器时实现了 Iterator trait 的对象。 该 trait 定义了一个next 方法，用于返回下一个元素。
    所有迭代器都实现了Iterator Trait， Iterator Trait定义与标准库，定义大致如下：

- type item 和 Self：：item 定义了与此该trait关联的类型；
    - 实现Iterator trait需要你定义一个Item类型，它用于next方法的返回类型（迭代器的返回类型）

关联类型

  • 关联类型时Trait中的类型占位符，它可以用于Trait的方法签名中
    • 可以定义出高喊某些类型的Trait，而在实现前无需知道这些类型是什么

关联类型与泛型的区别

  • 泛型
    • 每次实现Trait时标注类型
    • 可以为一个类型多次实现某个Trait（不同的泛型参数）
  • 关联类型
    • 无需标准类型
    • 不能为一个类型多次实现某个Trait

简单迭代器

  • number.iter() 创建了一个迭代器，该迭代器按顺序返回numbers向量中的每个元素
  • iter.next() 返回迭代器中的下一个元素，如果没有返回元素则返回None。

几个迭代的方法

  • iter方法：在不可变引用上创建迭代器
  • into_iter方法：创建的迭代器会获得所有权
  • iter_mut方法：迭代可变的引用

消耗迭代器的方法

  • 在标准库中，Iterator trait 有一些带默认实现的方法

  • 其中有一些方法会调用next 方法

    • 实现Iterator trait时必须实现next方法的原因之一

  • 调用next的方法叫做“消耗型迭代器”

    • 因为调用它们会把迭代器消耗尽

  • 例如 sum 方法（会耗尽迭代器）

    • 取得迭代器的所有权
    • 通过反复调用next，遍历所有元素
    • 每次迭代，把当前元素添加到一个综合里，迭代结束，返回总和

产生其他迭代器的方法

  • 定义在Iterator trait 上的另外一些方法叫做“迭代器适配器”
    • 把迭代器转换成不同种类的迭代器
  • 可以通过链式调用使用多个迭代器适配器来执行复杂操作，这种调用可读性较高。

map 方法

map 方法允许我们对迭代器中的每个元素应用一个函数，并返回一个新的迭代器

  • map 适配器将一个闭包应用于每个元素，这里是将每个元素乘以2

  • collect 消耗型适配器，将迭代器的结果收集到一个容器中，这里是 Vec

filter 方法

filter 方法允许我们根据条件筛选元素，并返回满足条件的元素的迭代器

  • filter 适配器根据条件筛选元素，这里是筛选出偶数
