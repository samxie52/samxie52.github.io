1. 创建 GitHub Pages 仓库

1.1 登录 GitHub

确保你已经登录到你的 GitHub 账户。

1.2 创建新仓库

  • 点击右上角的 “+” 号，选择 “New repository”
  • 或者直接访问 https://github.com/new

1.3 设置仓库名称

关键步骤：仓库名称必须严格按照格式：你的用户名.github.io

例如：

  • 如果你的用户名是 johnsmith，仓库名就是 johnsmith.github.io
  • 如果你的用户名是 alice123，仓库名就是 alice123.github.io

1.4 仓库设置

  • Public：必须设为公开仓库（GitHub Pages 免费版只支持公开仓库）
  • Initialize with README：可以勾选，方便后续操作
  • Add .gitignore 和 Choose a license：根据需要选择

1.5 创建仓库

点击 “Create repository” 完成创建。

7.3 配置 GitHub Pages

  1. 进入你的 username.github.io 仓库
  2. 点击 Settings 选项卡
  3. 滚动到 Pages 部分
  4. 在 Source 下选择：
    • Deploy from a branch
    • 分支选择：main
    • 文件夹选择：/ (root)

7.4 配置身份验证

方式1：Personal Access Token（推荐）

  1. GitHub → Settings → Developer settings → Personal access tokens → Tokens (classic)
  2. Generate new token → 选择 repo 权限
  3. 第一次部署时使用用户名和 token（不是密码）

常见问题排查

问题1：部署后网站不显示

  • 检查 GitHub Pages 设置中的分支是否为 main
  • 确认仓库名格式为 username.github.io
  • 等待几分钟，GitHub Pages 需要时间构建

问题2：本地预览正常，部署后样式异常

  • 检查 _config.yml 中的 url 和 root 配置
  • 确认所有资源文件路径正确

问题3：推送失败

  • 检查 Git 凭据配置
  • 确认使用正确的 Personal Access Token
  • 检查仓库权限
