如何实现 Native Staking

Method 2 - 使用 Solana CLI

Step-01: 创建质押账户

首先生成一个新的密钥对并创建质押账户：

参数说明：

  • --stake-authority: 有委托质押、拆分、合并质押账户权限的 key pair
  • --withdraw-authority: 有提现权限的 key pair

Step-02: 进行委托 (delegate)

委托说明：

  • <VOTE_ACCOUNT>: 目标验证器的投票账户地址
  • --stake-authority: 指定质押权限账户
  • --fee-payer: 指定交易费用支付账户
  • --with-memo: 添加交易备注信息

Step-03: 取消质押

当需要停止委托时，可以取消质押：

取消质押说明：

  • 取消质押需要等待一个完整的 epoch 才能生效
  • 在此期间质押仍然有效，会继续获得奖励
  • 取消生效后才能进行提现操作

Step-04: 提现 (withdraw)

质押取消生效后，可以提现 SOL：

提现说明：

  • <DESTINATION>: 接收 SOL 的目标钱包地址
  • <AMOUNT>: 提现的 SOL 数量
  • 只能提现已取消质押且冷却期结束的 SOL
  • 提现需要使用具有提现权限的账户

完整流程总结

  1. 准备阶段: 确保本地 Solana 环境正常运行
  2. 创建质押账户: 生成密钥对并初始化质押账户
  3. 选择验证器: 查看并选择合适的验证器
  4. 执行委托: 将质押委托给选定的验证器
  5. 取消质押: 当需要时可以取消委托（需等待 epoch 生效）
  6. 提现操作: 取消生效后提现 SOL 到指定地址
  7. 监控状态: 定期检查质押账户状态和收益

注意事项

  • 确保钱包中有足够的 SOL 用于创建质押账户和支付交易费用
  • 质押生效通常需要等待一个 epoch（约 2-3 天）
  • 取消质押也需要等待一个完整的 epoch 才能生效
  • 只有取消质押生效后才能进行提现操作
  • 选择验证器时要考虑其性能、佣金率和可靠性
  • 保管好相关的密钥文件，丢失将无法操作质押账户
  • 提现时需要指定准确的目标地址和金额
