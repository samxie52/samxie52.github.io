重要概念总结

  1. PDA (Program Derived Address):

    • 由程序 ID 和种子确定性生成的地址
    • 程序可以代表这些地址签名
    • 确保账户地址的唯一性和可预测性

  2. CPI (Cross-Program Invocation):

    • 允许一个程序调用另一个程序的功能
    • 本例中调用 Metaplex 程序创建元数据

  3. 账户验证:

    • Anchor 通过宏自动验证账户的有效性
    • 包括地址验证、权限检查、数据类型验证等

  4. 元数据集成:

    • 使用 Metaplex Token Metadata 标准
    • 提供丰富的 Token 信息（名称、符号、URI 等）

  5. 错误处理:

    • 当前错误可能由账户权限配置不当引起
    • 需要确保所有必需的程序都正确部署和配置

故障排除建议

如果遇到 “Cross-program invocation with unauthorized signer or writable account” 错误：

  1. 检查程序部署状态：确保所有依赖程序都已正确部署
  2. 验证账户配置：确保账户权限和可写性设置正确
  3. 使用 Devnet：考虑使用 Devnet 进行开发测试
  4. 调试账户信息：使用提供的调试脚本验证账户状态
