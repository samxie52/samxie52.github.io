智能指针2(Rc和RefCell以及Weak)

Rc - 引用计数指针

  • Rc 是 Reference Counted 的缩写，
  • 允许多所有者的共享所有权模型

RefCell - 内部可变性

  • 内留可变性:

    • RefCell允许你在其拥有的 T 内邮进行修改。即使 RefCell 本身是不可交的，这是通过在通行时进行范用借空实观的。

  • 运行时借用检查:

    • RefCell 使用动态借用检查，确保在运行时遵守 Rust 的借用规制。即，在任何时刻，RefCell 只能有一个可变借用，或者多个不可变借用，但不能同时存在。

  • borrow 和 borrow_mut 方法:

    • RefCell 提供了两个方法来独取对内部数捷的借用:
      • borrow():获取不可变借用(Ref)，可以同时有多个。
      • borrow_mut():获取可变借用(RefMut)。在同一时刻只能有一个。

关键点

  • 借用规则

    • RefCell 在运行时检查借用规则，以防止数据竞争来定义行为，编译器不进行这些检查，而是依赖RefCell的运行时检查。

  • 运行时开销

    • 因为RefCell需要在运行时检查借用规则，所以它会引入一定的性能开销，这在需要在编译时确定所有借用规则的场景中不可替代。

  • 错误处理

    • 如果违反了借用规则（例如，尝试同时获取多个可变借用），RefCell会在运行时引发 panic。

使用场景

  • 数据结构：

    • 在需要可变性但又受限于Rust的所有权系统时， RefCell 允许在数据结构中使用内部可变性。例如，实现需要共享修改的数据结构

  • 单线程环境

    • 在单线程环境中，RefCell 提供了一种在编译时无法保证借用规则的场景中使用内部可变性的方式。

引用循环与内存泄漏

在Rust中，引用计数(Rc)和原子引用计数(Arc)可以让多个所有者共享同一个数据。然而，这种共享机制如果不当使用，可能会导致引用循环(referencecycle)，从而造成内存泄漏。

Weak - 弱引用

Weak 通过 use std::rc::Weak 来引入，它具有以下特点:

  • 可访问，但没有所有权，不增加引用计数，因此不会影响被引用值的释放回收
  • 可由 Rc 调用 downgrade 方法转换成 Weak
  • Weak 可使用 upgrade 方法转换成 Option<Rc>，如果资源已经被释放，则 Option 的值是 None
  • 常用于解决循环引用的问题
