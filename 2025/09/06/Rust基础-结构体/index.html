普通结构体、元组结构体、单元结构体、方法、特征、trait

结构体

普通结构体

创建结构体实例

  • 初始化实例时，每个字段都需要进行初始化
  • 初始化时的字段顺序不需要和结构体定义时的字段顺序一致
  • 字段名使用逗号分隔
  • 初始化时，字段名和字段值之间用冒号分隔

修改结构体字段（可变性）

  • 整个实例必须是可变的
  • Rust并不允许只将结构体的某个字段设置为可变的

特殊的结构体

  • 元组结构体（Tuple Struct）
  • 单元结构体（Unit-like Struct）

元组结构体（Tuple Struct）

  • 结构体必须要有名称，但是结构体的字段可以没有名称，这种结构体长得很像元组，因此被称为元组结构体
  • 元组结构体在你希望有一个整体名称，但是又不关心里面字段时将非常有用
  • 例如 Point 元组结构体，

单元结构体（Unit-like Struct）

  • 如果你定义一个类型，但是不关心该类型的内容，只关心它的行为时，可以使用单元结构体
    • 定义 AlwaysEqual， 我们使用 struct 关键字，接着是我们想要的名称，然后是一个分号
    • 实例化不需要花括号或者圆括号

所有权

实现Copy特征的类型的所有权转移

  • 实现Copy特征的类型无需所有权转移，可以直接在赋值时进行数据拷贝

方法

定义方法

  • 使用 fn 关键字和名称声明，可以拥有参数和返回值，同时包含在某处调用该方法时会执行的代码
  • 方法与函数时不同的，因为它们在结构体的上下文中被定义，并且它们的第一个参数总是 self，它代表调用该方法的结构体实例

self | &self | &mut self

  • self 表示 Rectangle 的所有权转移到该方法中，这种形式比较少
  • &self 表示该方法对 Rectangle 的不可变借用
  • &mut self 表示该方法对 Rectangle 的可变借用

方法名跟结构体字段名相同

  • 在 Rust 中，方法名可以与结构体字段名相同

关联函数

  • 所有在 impl 块中定义的函数被称作关联函数 （associated functions），因为它们与 impl 后面命名的类型关联
  • 关联函数的第一个参数不需要 self，因为它们不操作结构体的实例
  • 因为是函数，所有不能用 . 来调用，而是用 :: 来调用
  • Rust中有一个约定俗成的规则， 使用 new 作为构造函数的名称，出于设计上的考虑，Rust 语言设计者没有为 new 提供任何特殊语法支持

多个impl定义

  • Rust 允许为一个结构体定义多个 impl 块，目的是提供更多的灵活性和代码组织性。
  • 例如，我们可以为 Rectangle 定义多个 impl 块，一个用于计算面积，另一个用于设置宽度。

实现trait

为类型实现特征

  • 如果不同的类型具有相同的特征，那么我们就可以定义一个特征，然后为这些类型实现该特征
  • 定义特征是把一些方法组合在一起，目的是定义一个实现某些目标所必须的行为的集合
